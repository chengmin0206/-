## 一、需求回顾（1分钟）

### 产品定位
卡伴日记是一款面向年轻人群的**健康管理工具**，通过饮食记录、体重追踪、AI健康助手等功能，帮助用户养成健康的生活习惯。

### 目标用户
- 18-35岁年轻白领、大学生
- 有减肥或健康饮食需求的用户
- 希望科学管理身体的用户

### 核心需求
1. 方便快捷地记录每日饮食
2. 追踪体重变化和健康指标
3. 获取个性化的健康建议
4. 享受丰富的健康食谱资源

### （四）交互流程

**已完成的流程图**：✅ 5个核心流程

#### 1. 用户登录流程
```
启动页 → 账户创建页 → 选择登录方式
    ├─ 手机号 → 输入手机号 → 获取验证码 → 验证成功 → 进入应用
    ├─ 一键登录 → 调用SDK → 进入应用
    ├─ 第三方 → 授权 → 进入应用
    └─ 访客 → 直接进入应用（功能受限）
```

#### 2. 饮食记录流程
```
饮食页面 → 点击添加 → 选择餐次 → 输入食物 → 自动计算营养 → 保存
```

#### 3. 体重记录流程
```
体重页面 → 点击添加 → 输入数据 → 自动计算BMI → 保存 → 更新趋势图
```

#### 4. AI助手对话流程
```
聊天页面 → 输入问题 → 发送 → 显示用户消息 → AI处理 → 显示回复
```

#### 5. 食谱浏览流程
```
食谱中心 → 浏览分类 → 查看详情 → 收藏/创建
```

### 方案调整说明

**实践二方案调整**：

原计划中，我们计划使用微信小程序作为开发平台。但由于以下原因，我们调整为使用H5开发：

1. **技术限制**：微信小程序对AI调用的限制较多，功能实现受限
2. **部署问题**：小程序发布审核周期长，影响演示进度
3. **学习成本**：团队成员更熟悉Web开发

**调整内容**：
- 开发平台：微信小程序 → H5 + 后端
- 部署方式：小程序商店 → 可独立运行的Web应用
- 用户注册：微信授权 → 支持多种登录方式

**调整后的优势**：
- 开发更灵活，不受平台限制
- 可以随时部署和演示
- 更容易实现AI功能对接
